# Week 2.5 Chunk优化实验报告

## 1. 实验目标
验证不同chunk策略对检索性能的影响

## 2. 实验设计
- 4种策略：300/500/800/qa_split
- 996条Stack Overflow数据
- 20个混合难度测试queries
- 评估指标：Recall@5, MRR

## 3. 实验结果
| 策略 | Recall@5 | MRR | Hard难度 |
|------|----------|-----|----------|
| fixed_300 | 0.800 | 0.800 | 0.700 ✅ |
| fixed_500 | 0.750 | 0.688 | 0.600 |
| fixed_800 | 0.800 | 0.692 | 0.600 |
| qa_split | 0.800 | 0.702 | 0.600 |

## 4. 关键发现
1. 小块（300）在困难query上表现最好
2. 大块（800）在中等难度完美但损失精确度
3. 问答分离对完整问题好，但对碎片化query不够robust
4. 中等块（500）陷入"中间地带"

## 5. 结论
推荐使用fixed_300策略（chunk_size=300, overlap=30）
